# Governance

## Roles

- **Primary Maintainer**: owns releases, security response, roadmap execution.
- **Backup Maintainer(s)**: approves breaking changes, can execute continuity handoff.
- **Contributors**: propose changes through issues/PRs.

## Decision Process

1. Open issue/RFC for non-trivial changes.
2. Collect ecosystem impact and security implications.
3. Merge with maintainer approval quorum (2 approvals for breaking changes).

## Public-Goods Principles

- Keep core components permissively licensed.
- Prioritize interoperability with broader Stellar ecosystem.
- Publish transparent milestone and budget reporting.
