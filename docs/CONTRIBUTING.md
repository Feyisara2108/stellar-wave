# Contributing

## Workflow

1. Open or reference an issue.
2. Create a focused branch.
3. Add tests for behavior changes.
4. Submit PR using the template.

## Standards

- Rust: `cargo fmt`, `cargo clippy`, `cargo test`.
- TypeScript: strict mode (`tsconfig`), `npm run --prefix sdk build`.
- Keep contract storage schema changes documented.

## DCO

By contributing, you certify Developer Certificate of Origin sign-off on commits.

## Maintainer SLAs

- Issue triage target: `<48h`
- PR first response target: `<7d`
- Security issues: acknowledge within 24h
